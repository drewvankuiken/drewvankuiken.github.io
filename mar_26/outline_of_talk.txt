Section: Entry model
14. Timing
	⁃	A random number of projects of each technology type appear. Enter or die. If enter, choose county
	⁃	The grid operator views last period congestion, makes upgrade decisions
	⁃	Production occurs for h=4,380 hours in t=half-year
	⁃	Not sure if there will be exit yet, need to review how much exit happens in the data
15. Production & Grid Upgrades
	⁃	Generators aggregated to technology level
	⁃	Each technology submits marginal costs to grid operator, grid operator runs dispatch model
	⁃	Grid operator runs grid upgrade model
	⁃	Note: once entered, firms make no decisions currently
16. Entry
	⁃	N_{\tau} entrants show up each period, where N_{\tau} is fixed to the max number of counties of any technology type entered in each half-year + 1
	⁃	Entering incurs fixed cost \nu_{r\tau t} (q) + \varepsilon_{r\tau t} (unobserved fixed costs)
	⁃	Entry depends on fixed costs and expectations about state transitions. States include: local capacity of each technology type, grid-wide capacity of each technology type, capacity of key constraints
	⁃	Implicitly profits depend on how integrated your region is: higher k means you compete against more firms, earn system lambda. Higher q means congestion pricing hits, lower profits.
17. Equilibrium
	⁃	Entrants track own region state variables q_{r\tau t}, k^l_{rt}. Assume k solves according to own region quantity and aggregate quantities only.
	⁃	Entrants assume aggregate quantities evolve according to AR-1 process
	⁃	Entry forms a moment based Markov equilibrium (MME, Ifranch & Weintraub 2018)
18. Condensing the state space
	⁃	MME helps with curse of dimensionality
	⁃	Still ~80 counties with substantial generation, 300 transmission corridors
	⁃	Proposal: using most frequently congested lines, condense generation into ~40 regions, connected by X transmission lines, let grid evolve for each region
	⁃	Problem parallelizable, which is helpful
